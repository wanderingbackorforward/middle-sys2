## 目标
- 通过 Chrome DevTools MCP Server 在本地已登录的 Chrome 中自动打开哔哩哔哩“动态”，填写“早上好”并发布。

## 前置条件
- 已在 Chrome 中安装并启用 Chrome DevTools MCP Server。
- 选择要使用的 Chrome 用户配置文件，并确保该配置文件已登录哔哩哔哩。
- 允许 MCP Server 与助手连接（授权一次）。

## 连接与权限
- 连接到 MCP Server，列出并验证可用工具（导航、DOM 查询、点击、输入、等待、截图）。
- 将作用域限制到 `https://t.bilibili.com/*` 与相关静态资源域，最小权限原则。

## 自动化流程（MCP 命令序列）
- 打开页面：导航到 `https://t.bilibili.com/`。
- 等待登录：
  - 通过检测登录状态元素或检查页面是否出现“登录”入口；如未登录，暂停并提示你在同一标签页完成登录后继续。
- 定位输入框：
  - 使用选择器策略逐步查找动态编辑区（优先 `textarea` 或 `div[contenteditable="true"]`）。
  - 容错：若选择器失效，尝试通过包含提示文案（如“说点什么”）的邻近节点回退定位。
- 输入文案：
  - 在编辑区输入“早上好”。
- 点击发布：
  - 查找并点击“发布”按钮（按钮文本或具备提交语义的按钮）。
- 等待成功：
  - 监听 DOM 变化出现新动态条目；或监听网络请求完成状态。
- 截图与验证：
  - 截取页面含新动态的区域，保存本地以便确认。

## 选择器与稳健性
- 首选稳定类名或结构（如“发布/发动态”按钮文本）。
- 设定超时与重试策略（3 次，每次 2–5 秒间隔）。
- 若页面存在弹窗（如发布引导/权限弹窗），尝试关闭后继续。

## 验证标准
- 成功条件：时间为当前的新动态，正文含“早上好”。
- 验证方法：刷新页面确认；可选再次进入“我的动态”页面查看。

## 安全与隐私
- 整个流程在你的本地浏览器内执行，不采集或传输账号与 Cookie。
- MCP 仅对目标页面执行必要的 DOM 操作与导航。
